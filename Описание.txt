Для корректной установки нашего веб приложения нужно выполнить следующее:
1.	Запустить скрипты с расширение «.sql» для создания базы данных и необходимых  для работы процедур.
2.	В файле Web.config в строке <stringConnection> прописать параметры для корректного подключения к базе данных.

Описание для пользователя
Данное веб приложение реализовано для учета корневых и розничных продавцов, а также их фирм.
В нем реализован  следующий функционал
- добавление как корневого продовца, так и дочернего,
- добавление новых предприятий, как корневого продовца, так и дочернего,
- получение всех розничные продавцы в древовидной структуре
- получение корневого розничного продавца для указанного
- получение списка действительных контрактов для выбранного розничного продавца, а также для его дочерних продавцов если они есть
- создавать новый договор между несколькими розничными сетями
- получение списка всех контрактов с использованием выгружаемого вывода с возможностью применения фильтра к выбранным критериями розничным продавцам, выбранной дате подписания контракта или диапазону дат, действительного или неактивного контракта (Все, Действительный, Неактивный). Фильтр работает по логике "И".
При запуске проекта на главной странице отображается список с корневыми продовцами. Если кликнуть на одно, то выпадут его предприятия, если кликнуть на предприятие выпадит список дочерних продавцов. И так пока данные не закончатся.
Сверху отображены весь каталог страниц проекта.
Также при отображению контрактов по фильтру диапазон дат нужно вводить в формате “dd.MM.yyyy”, реализовано проверку на формат, если ввести некорректную дату то запрос на сервер не  будет отправлен.
